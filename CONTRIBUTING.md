# Contributing to Gitto

Thank you for your interest in contributing to Gitto!

## Development Setup

1. Fork the repository
2. Clone your fork
3. Set up development environment (see README.md)
4. Create a feature branch

## Code Style

### Python
- Follow PEP 8
- Use type hints
- Document functions with docstrings
- Maximum line length: 100 characters

### TypeScript/React
- Use TypeScript strict mode
- Follow React best practices
- Use functional components with hooks
- Maximum line length: 100 characters

## Testing

- Write tests for new features
- Ensure all tests pass before submitting PR
- Add tests for edge cases
- Update test documentation if needed

## Pull Request Process

1. Update README.md if needed
2. Update CHANGELOG.md with your changes
3. Ensure all tests pass
4. Request review from maintainers
5. Address review feedback

## Commit Messages

Use clear, descriptive commit messages:
```
feat: Add trust report endpoint
fix: Fix conservation proof calculation
docs: Update API documentation
test: Add metamorphic tests for reconciliation
```

## Questions?

Open an issue or contact the maintainers.
